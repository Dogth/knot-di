\doxysubsection{Подробное описание}
Контейнер для управления сервисами 

Этот класс предоставляет функциональность для регистрации и разрешения сервисов, а также управления их жизненным циклом. Он поддерживает стратегии синглтона и временных сервисов, а также использует \doxylink{class_knot_1_1_memory_pool}{Memory\+Pool} для управления памятью.

Получение зарегистрированного сервиса по его типу 

Этот метод позволяет получить зарегистрированный сервис по его типу. 
\begin{DoxyTemplParams}{Параметры шаблона}
{\em T} & Тип сервиса, который нужно получить \\
\hline
\end{DoxyTemplParams}
\begin{DoxyReturn}{Возвращает}
Указатель на сервис типа T, или nullptr, если сервис не зарегистрирован или не может быть создан.
\end{DoxyReturn}
\begin{DoxyNote}{Заметки}
Если сервис зарегистрирован как SINGLETON, он будет создан при первом вызове resolve и сохранен для последующих вызовов. Если сервис зарегистрирован как TRANSIENT, он будет создан каждый раз при вызове resolve. 
\end{DoxyNote}
