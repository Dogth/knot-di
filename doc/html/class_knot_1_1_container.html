Контейнер для управления сервисами

Этот класс предоставляет функциональность для регистрации и разрешения сервисов, а также управления их жизненным циклом. Он поддерживает стратегии синглтона и временных сервисов, а также использует MemoryPool для управления памятью.

Конструктор контейнера

Создает контейнер с нулевым счетчиком сервисов и временных сервисов, а также инициализирует пул памяти с размером 4096 байт.

Заметки
Если требуется использовать другой размер пула памяти, рекомендуется использовать конструктор с параметром max_bytes.

Регистрация сервиса в контейнере

Этот метод позволяет зарегистрировать сервис в контейнере с заданной стратегией создания (SINGLETON или TRANSIENT).

Аргументы
strategyСтратегия создания сервиса
Параметры шаблона
TТип сервиса, который нужно зарегистрировать
Возвращает
true, если регистрация успешна, иначе false.
Заметки
Если сервис уже зарегистрирован, регистрация не будет выполнена.

Получение зарегистрированного сервиса по его типу

Этот метод позволяет получить зарегистрированный сервис по его типу.

Параметры шаблона
TТип сервиса, который нужно получить
Возвращает
Указатель на сервис типа T, или nullptr, если сервис не зарегистрирован или не может быть создан.
Заметки
Если сервис зарегистрирован как SINGLETON, он будет создан при первом вызове resolve и сохранен для последующих вызовов. Если сервис зарегистрирован как TRANSIENT, он будет создан каждый раз при вызове resolve.
